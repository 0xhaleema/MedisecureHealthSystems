# 🏥 Information Assurance Plan — MediSecure Health Systems (MHS)

A comprehensive **Information Assurance (IA) planning project** for a fictional mid-sized healthcare organization — MediSecure Health Systems. This project covers the full IA lifecycle from asset identification and risk assessment to security policies, contingency planning, and financial justification.

> 📄 **Note:** This is a documentation-only project. No code is included.

---

## 🏢 Organization Profile

| | |
|---|---|
| **Organization** | MediSecure Health Systems (MHS) |
| **Type** | Healthcare Support Services |
| **Size** | 200 employees, ~300 endpoints |
| **Revenue** | ~$25–30 million/year |
| **Locations** | 1 Head Office + 1 Operations Center |
| **Infrastructure** | On-premise servers + cloud-hosted applications |

MHS provides non-clinical healthcare support including employee data management, patient booking systems, billing and insurance processing, and medical records management.

---

## 📋 Project Coverage

### 1. 🏛️ Governance & Organization
- Executive roles and responsibilities (CEO, CFO, CISO, CIO, IT Manager)
- Organizational chart and governance structure
- Regulatory compliance: **HIPAA**, ISO/IEC 27001

### 2. 🗃️ Asset Identification
- Information, physical, software, and human assets
- Critical asset analysis with CIA requirements and business impact

### 3. 💰 Cost-Benefit Analysis
| | Amount |
|---|---|
| Annual IA Investment | $700,000 |
| Expected Annual Loss (without IA) | $1,005,000 |
| Net Annual Benefit | $305,000 |
| **ROI** | **43.6%** |

### 4. ⚠️ Risk Assessment
7 threats identified and scored using **Likelihood × Impact** matrix:

| Risk | Level |
|---|---|
| Ransomware Attack | 🔴 High (9) |
| Data Breach | 🔴 High (6) |
| Phishing Attack | 🔴 High (6) |
| Insider Threat | 🔴 High (6) |
| System Failure | 🟡 Medium (3) |
| Power Outage | 🟡 Medium (3) |
| Natural Disaster | 🟡 Medium (3) |

Includes heat map, vulnerability mapping, and mitigation controls per risk.

### 5. 📜 Security Policies
Six policies defined, aligned with ISO/IEC 27001/27002 and NIST standards:
- Information Security Policy
- Access Control Policy (RBAC, MFA)
- Data Classification Policy
- Acceptable Use Policy
- Incident Response Policy (ISO/IEC 27035)
- Backup and Recovery Policy (NIST SP 800-34)

### 6. 🌐 Network Architecture
- Designed network diagram for MHS infrastructure
- Covers on-premise and cloud-hosted components

### 7. 🔄 Contingency Planning
| | |
|---|---|
| **RTO** | 6 hours max downtime |
| **RPO** | 2 hours max data loss |
| **Backup** | Daily incremental + weekly full, encrypted off-site cloud |
| **Recovery Site** | Cloud-based warm site |

Includes manual fallback procedures, communication plan, and staff roles during incidents.
